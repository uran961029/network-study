네트워크에 연결되는 기기나 단말기가 고유하게 가지는 식별자.
IEEE(미국전기전가학회)가 관리하고, 전 세계 기기 제조사에 고유 코드를 할당 함.

1. 주소의 구조
	1. 컴퓨터의 네트워크 카드, 스마트폰, 라우터, 스위치 등 네트워크에 연결되는 모든 기기에 하나씩 부여되는 식별자.
	2. 48비트 길이의 식별자이며 이 중 상위 24비트를 벤더코드(OUI식별자)로 장비 제조사에 할당하고 제조사는 여러개의 벤더 코드를 가질 수 있음.
	3. 하위 24비트(노드 번호)는 각 제조사의 책임하에 제품에 하나씩 할당 됨.
2. MAC 주소와 IP주소의 차이
	1. 둘 다 프로토콜에 사용되는 식별자라는 점에서 동일하지만, 사용되는 계층이 다름
	2. MAC 주소는 이더넷 프레임?에서 장치나 단말기 식별자로 사용되며, 이더넷은 2계층이므로 MAC주소는 OSI 참조 모델에서 2계층(이하)의 식별자.
	3. IP 주소는 TCP/IP 프로토콜에서 출발지나 목적지의 장치나 네트워크 식별자로 사용되며, TCP/IP나 인터넷은 3계층 이상의 계층이므로 IP주소는 3계층 이상의 식별자.
3. ARP
	1. Address Resolution Protocol, IP 주소에서 해당 장비의 MAC 주소를 확인하는 프로토콜. IP 주소 정보를 전송하고 특정 대상과 통신하기 위해 질의함. 
	2. 장치 간 통신에서는 MAC주소가 필요하지만, MAC 주소는 2계층 이하이므로 애플리케이션은 IP주소를 알아도 MAC주소는 모름. 따라서 LAN 내에서 패킷을 교환하려면 해당 IP 주소가 LAN 내 어느 MAC 주소를 가진 장치의 것인지 확인하고 이더넷 프레임을 구성해야 함. 이때 사용되는것이 ARP
	3. 작동 방식 
		1. ARP는 MAC 주소를 알고자 하는 IP 주소 정보를 LAN에 대상을 지정하지 않고 전송(LAN의 모든 장치에 대상을 지정하지 않고 패킷을 전송하는 것을 브로드캐스트라고 함. )
		2. 브로드캐스트 패킷을 모든 장치가 수신하여, 해당 장치들은 자신의 MAC 주소를 회신함. (유니캐스트-1:1전송)
		3. 실제로 모든 통신마다 ARP로 조회하는것은 비효율적이라 허브나 라우터에 LAN 내 모든 기기의 IP/MAC 주소의 대응표(주소 테이블)를 만들어 둘 때가 많음.
