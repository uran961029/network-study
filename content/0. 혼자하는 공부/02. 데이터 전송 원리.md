
1. 데이터 전송 방식
	1. 직렬 전송
		- 하나의 신호선으로 펄스형 신호를 보내는 방식
		- 한번에 1비트의 데이터를 보낼 수 있음
	2. 병령 전송
		- 8개나 16개처럼 여러개의 신호선을 사용하여 한번에 8비트나 16비트 잔위로 데이터를 보낼 수 있음.
		- 여러개의 신호선을 사용하기때문에, 장거리를 연결에는 부적합 > 따라서, 현재 통신 네트워크 대부분은 직렬 전송을 사용함.
	3. 직렬 전송은 이론적으로 신호선 하나로 양방향 통신이 가능하나, 보통은 송/수신용 신호신을 따로 두어 총 2개를 사용함. 

2. 비동기 통신과 동기 통신
	1. 통신의 시작과 끝에는 특별한 신호/약속이 필요함. 이게 없으면 데이터를 수신할때 어디부터 어디까지가 데이터인지 알 수 없음. 
	2. 시작하는 신호를 시작비트 / 종료하는 신호를 종료비트라고 함. 
	3. 비동기 통신 : 시작과 종료 신호를 사용하는 방식
	4. 동기 통신 : 클럭이라는 동기신호를 사용해서 데이터를 보내는 방식 
		- 클럭 신호 : 일정한 간격으로 보내는 신호. 신호와 데이터는 클럭 신호의 펄스에 맞추어 전송됨. 

3. 포로토콜
	1. 프로토콜의 정의
		1. 프로토콜에는 통신할 데이터의 구성, 데이터를 확실하게 주고받는 절차 등 각종 결정 사항과 기준이 정의 되어 있음.
		2. 네트워크에서 데이터를 주고받으려면 송신 측 / 수신 측이 서로를 인식하고 있어야 함.
		3. 그저 흐르고 있을 뿐인 데이터를 어느 부분이 목적지 데이터인지, 어디가 신호인지, 어디가 데이터 본체인지에 대한 약속이 필요하며 이 약속이 프로토콜. 
		4. 같은 프로토콜을 사용하면 다른 기기간에도 통신이 가능하며, 프로토콜이 다르면 같은 기기끼리도 서로 통신할 수 없음.
		5. 프로토콜은 보통 표준화 단체나 컨소시엄등에서 논의와 검증을 거쳐 결정함. 
	
	2. 주요 프로토콜
		1. IEEE 802.3 : IEEE가 규정하는 이더넷 규격의 프로토콜. 랜 케이블 등에 사용됨
		2. IEEE 802.11 : IEEE가 규정하는 무선 LAN 규격의 프로토콜. WI-FI로 보금되어 있음.
		3. IP : 여러 네트워크를 연결하는 프로토콜. 인터넷을 구성함
		4. ARP : Adress Resolution Protocol, IP주소에서 각 기기가 가지고 있는 MAC 주소를 알아내는 프로토콜
		5. TCP : Transmission Control Protocol, 인터넷 등에서 상위 계층과 신뢰성을 높여 통신하는 프로토콜
		6. UDP : User Datagram Protocol, 인터넷 등에서 상위 계층과 속도와 편의성을 우선하여 통신하는 프로토콜
		7. POP : Post Office Protocol, 이메일 수신 프로토콜
		8. SMTP : Simple Mail Transfer Protocol, 이메일 송신 프로토콜
		9. HTTP : HyperText Transfer Protocol, 웹 브라우저와 웹 서버 간 통신 프로토콜
		10. DNS : Domain Name System, DNS 서버에 IP 주소를 문의해서 도메인 이름을 IP 주소로 변환하는 포로토콜
