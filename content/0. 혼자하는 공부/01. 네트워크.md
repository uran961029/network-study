
1. 네트워크의 의미
	1. 기술적 관점에서 바라본 네트워크
		1. 어떤 대상과 다른 대상을 특정한 장치나 규칙으로 연결한 유기적 시스템
		2. 사람/사물들도 그 대상에 포함되지만, 기술적 관점에서는 컴퓨터/스마트폰 등 기기나 시스템을 의미함. (예시로, 인터넷, 휴대전화망, 은행ATM 등)
	2. 네트워크에서는 연결되는 대상을 노드라고 함. 
	3. 따라서 네트워크는 노드의 연결을 의미! >> 네트워크라고 부르려면 노드가 반드시 다른 하나 이상의 노드와 연결되어 있어야 함.
		1. 노드를 연결하는 방식을 네트워크 토폴로지라고 함
		2. 각 노드는 반드시 어떤 노드와 선(유선/무선)으로 연결되어 있으며, 분리 독립된 단일노드는 존재하지 않음
		3. 기능/연결방식/역할등이 네트워크 종류와 구조를 결정
		4. 네트워크는 꼭 하드웨어에만 국한되지 않음. 예를들어 AI 개발에 이용되는 신경망이나 폴더의 계층 구조를 나타내는 디렉터리 트리도 네트워크라고 할 수 있음

2. 네트워크의 연결 형태와 종류
	1. 네트워크 토폴로지란?
		1. 네트워크는 노드의 연결로 구성되며, 이 연결 방법이나 형태를 네트워크 토폴로지라고 함. 예를 들어, 일대일 / 다대다/ 일대다 등 연결하는 상대의 수나 연결 방법 등에 따라 몇가지로 분류할 수 있음
	2. 네트워크 토폴로지의 종류
		1. 라인형 : 각 노드가 선형으로 연결
		2. 링형 : 각 노드가 링 형태로 연결
		3. 스타형 : 허브라는 기점노드에 모든 노드가 연결
		4. 버스형 : 모선 한줄에 모든 노드가 연결
		5. 트리형 : 각 노드가 부모 or 자식 노드에 연결
			- 루트 노드 : 부모가 없는 노드
			- 리프(잎) 노드 : 자식이 없는 노드
		6. 메시형 : 각 노드가 하나 이상의 노드와 임의로 연결. 그 중 모든 노드가 모든 노드와 직접 연결된 것을 풀 메시(풀 커넥트)라고 함.
	3. 복합형 네트워크
		1. LAN처럼 소규모 네트워크가 아닌이상, 복합형 토폴로지를 사용하는게 일반적.
		2. 예를 들어서 휴대전화망은 기지국을 허브로하는 스타형이지만, 기지국끼리는 메시형 or 라인형 등이 섞여있음
		3. 또, 네트워크를 관리하는 방법이 물리적 연결방법과 반드시 일치하지는 않음. 
		4. 예를 들어서 인터넷을 구성하는 라우터는 메시형이지만, 웹상의 도메인 이름은 루트네임 서버를 정점으로 하는 트리형임 (이 부분 헷갈림)

3. 네트워크 구성 요소
	1. 서버, 라우터, 허브, 케이블 등 물리장비들이 꼭 필요함. 각 기기를 허브로 연결하는 것이 기본적인 네트워크라고 할 수 있음.
	   
	2. 하드웨어 요소(LAN을 기준으로)
		1. 각 기기를 연결하려면 허브를 이용해서 네트워크 신호선끼리 접속을 확보해야함. 
		2. 현재 개념의 허브는 단순히 커넥터를 이어주는 것 뿐 아니라, 네트워크상의 데이터를 해석해서 올바른 대상만 연결해 줌 (스위칭 허브라고도 함.)
		3. 가장 기본적인 LAN은 기기가 허브로 연결된 네트워크를 의미하며, 이를 세그먼트라고 함.
		4. 여러개의 허브로 하나의 LAN을 구성할 수도 있으며, LAN끼리 연결할 때는 라우터를 이용함. 라우터는 LAN끼리 연결하는 기기!
		5. 라우터는 수신한 신호가 자신이 관리하는 LAN 앞으로 온것인지 판단 후 데이터를 LAN 내부로 받아들이거나 다른 라우터에 전달함. 자신이 관리하는 LAN의 내부 통신은 외부로 보내지 않고 LAN 내부에서 처리 / 외부로 나가는 통신은 다른 라우터에 전송하여 처리를 넘김
	
	3. 주요 장치 및 도구
		1. LAN 케이블
		2. 스위칭 허브(스위치와 같은 것?) : 네트워크 신호를 중계하는 장치. 여러 기기의 LAN 케이블을 연결할 수 있으며, 지정된 노드에만 신호를 보냄. 스위치로 연결된 범위가 네트워크의 최소 단위임. 
		3. 리피터 : 스위치와 비슷하지만 수신한 신호를 그대로 전송하며, 일반적으로 신호를 바로잡고 증폭해서 접속거리를 늘릴 수 있다. 
		4. 브리지 : 프로토콜이나 전송 원리가 다른 네트워크 사이에서 신호를 중계함. 신호레벨, 변조 방식, 프로토콜 등을 변환 함. 
		5. 네트워크 인터페이스 카드 : 네트워크 신호와 컴퓨터 내 신호를 변환하는 회로 기판.
		6. 라우터 : 스위치로 묶인 네트워크(세그먼트)끼리 연결하는 장치. 
		7. 게이트 웨이 : 네트워크에 연결하는 접속점. 같은 프로토콜 내부에서 네트워크나 세그먼트 경계에 설치되어 패킷 출입을 제어함. 특정 기술을 가리키는 용어는 아니며, 라우터나 서버 등 네트워크 장치라면 게이트웨이로 정의가능.
