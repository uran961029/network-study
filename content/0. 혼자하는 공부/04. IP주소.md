IP 주소는 출발지나 목적지를 식별하는 주소로 사용되는 데이터.
인터넷에서 사용하는 글로벌 IP주소와 LAN 내에서 사용하는 프라이빗 IP주소가 있음.

1. IP 주소의 구조
   IPv4와 IPv6라는 두가지 버전이 있음. 
	1. IPv4
		1. TCP/IP 프로토콜에서 출발지나 목적지를 식별하는 식별자로 사용되는 데이터
		2. 32비트로 구성된 데이터로, 32비트를 8비트씩 4등분 > 8비트단위를 네트워크에서 옥텟이라고 함.
		3. 32비트면 주소를 약 43억개 표현할 수 있으므로 컴퓨터나 장치를 약 43억대 가량 인터넷에 고유하게 연결할 수 있음. 하지만 실제로 주소 공간을 43억개 모두 사용할 수 있는것은 아님. (예약된 특수 주소 공간 및 프라이빗 IP 주소공간은 사용X)
	2. 글로벌 IP주소 & 프라이빗 IP주소
		1.  IP주소의 주소공간 중 인터넷 주소로 사용할 수 있는 주소를 글로벌 IP주소라고 하며, 이 주소를 가진 기기는 인터넷상에 하나 뿐.
		2. 인터넷에 직접 연결되지 않고 LAN 내부 등의 주소 공간에서 사용하는 IP 주소를 프라이빗 IP주소라고 함. 이 주소는 인터넷에서 사용하지 않으므로 LAN마다 중복되지 않게만 관리하면 다른 LAN에서 같은 IP주소를 사용해도 문제 없음
	3. IPv6
		1. IPv4 주소가 점점 고갈되면서 새로운 IP 주소 체계가 필요하게 되자 책정된 것
		2. 128비트, 즉 16옥텟으로 IP주소를 나타냄. 
		3. 프로토콜 수준에서 암호화 통신이 규정되어 있으며, MAC 주소를 이용하여 IPv6 주소를 자동으로 생성할 수 있고 2계층을 포함한 설정 자동화가 가능함. 
		4. 헤더가 IPv4보다 단순해서 처리가 간단.
		5. 계층 구조가 엄격해서 라우팅 효율이 좋음
		6. 다만 두 버전의 프로토콜 호환성이 없어서 IPv6로 인터넷에 연결하려면 IPv6와 호환되는 앱이나 서비스를 사용해야 함.
