네트워크는 일반적으로 계층 구조로 구성 됨. 서브넷은 하나의 네트워크를 여러 세그먼트로 나누는 것을 의미 함. 어떤 네트워크에 대해 하나하나의 세그먼트를 서브넷이라고 한다.

1. 네트워크 분할 방식
	1. 일반적인 LAN은 장비의 종류나 용도, 사용 부서 등에 따라 세그먼트로 분할 됨. IP 네트워크에서는 IP 주소 상위 1비트부터 5비트까지 패턴에 따라 주소 영역을 클래스로 나누어 생각함. (A~E)
	2. 그러나 클래스 방식은 낭비가 심해 IP 주소의 임의의 비트 위치에서 클래스 부분(네트워크부)과 주소 부분(호스트부)으로 구분하는 CIDR 방식이 사용 됨.
	3. 네트워크부는 IP 주소 상위에서 임의의 비트 수를 사용해서 그 네트워크를 대표하는 주소이며, 호스트부는 해당 네트워크 내에서 사용할 수 있는 개별 주소 범위.
	4. CIDR 방식으로 분할한 네트워크가 서브넷
		1. 서브넷의 네트워크부와 호스트부 구분은 서브넷 마스크로 나타냄.
		2. 네트워크부의 비트 패턴을 모두 1(10진수의 255)로 하고 나머지를 모두 0으로 한 주소 표기가 서브넷 마스크. 
		3. 다만 개별 호스트의 IP 주소만 봐서는 서브넷 구분이 몇 비트째인지 알 수 없으므로 IP주소 뒤에 슬래시와 네트워크부의 비트 수를 붙인 프리픽스 표기로 나타냄(예시: 192.168.100.20/24)
