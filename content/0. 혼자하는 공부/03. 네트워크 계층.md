1. 계층별로 나누어 처리하기
	1. 애플리케이션단에서 데이터를 정확하게 전송할 수 있도록 프로그램이 되어 모든 변환 처리를 수행하는것도 가능하지만, 프로토콜, OS 등에 대한 변환까지 모두 애플리케이션이 수행한다면 프로그램은 복잡해지고 낭비가 많아짐.
	2. 따라서 네트워크를 하드웨어, 프로토콜, 애플리케이션이라는 레이어에서 각각 분담하여 처리하도록 나눠버림.

2. 프로토콜 스택
	1. 네트워크에서는 다양한 프로토콜이 사용되는데, 프로토콜의 차이는 프로토콜 스택이라는 계층 구조를 이용해서 처리함.
	2. 프로토콜 = 헤더 + 페이로드
		1. 헤더 : 프로토콜 정보(우편으로 치자면 주소 등)
		2. 페이로드 : 데이터 그 자체
	3. 하위계층 프로토콜의 페이로드에 상위 계층의 헤더와 페이로드를 포함시키면, 해당 프로토콜 계층은 자신의 헤더 정보만 해석해서 페이로드를 상위 네트워크에 전달함. 페이로드는 상위 계층의 헤더와 페이로드로 되어있기때문에 상위 네트워크의 프로토콜이 제대로 처리될 수 있음. (39p 이미지 참조)

3. 네트워크 모델
	1. OSI(Open Systecms Interconnection) 참조 모델
		- ISO(국제표준화기구)에서 제정한 계층화 된 네트워크의 대표적 통신 모델
		- 1977년에 제정되었으며, 당시 네트워크 기술을 기반으로 하기에 현재는 OSI 참조 모델을 그대로 구현한 네트워크나 장비는 거의 없음.
		- 7계층으로 나뉘어 있는 것이 특징
		- 포로토콜을 계층 구조로 만든 개념 : 애플리케이션은 연결되는 컴퓨터나 케이블의 차이를 처리할 필요가 없으며, 네트워크 구성 요소가 계층으로 분리되므로 시스템을 프로토콜 스택 개념으로 설계하고 개발할 수 있음.
	2. TCP/IP 모델
		- 인터넷을 기준으로 한 네트워크 통신 모델

4.  TCP/UDP/IP
	1. 인터넷에서 통신할 때 사용하는 기본 프로토콜.
		1. IP 프로토콜 : UDP, TCP, HTTP 등 다른 프로토콜의 기본이 됨.
			- IP 주소를 참조하여 페이로드를 올바른 목적지에 전송하는 최소한의 절차와 규칙을 규정.
			- 페이로드에는 UDP/TCP 뿐 아니라 상위 계층의 프로토콜 데이터가 저장됨. 
		2. UDP/TCP 프로토콜 : 식별자로, 각각 헤더에 포트 번호를 부여해서 IP 주소가 나타내는 사용자나 애플리케이션 종류등을 지정
			- UDP 프로토콜 : 포트 번호를 부여할 뿐 처리 과정 자체는 IP 프로토콜과 거의 다르지 않음. 확인이나 재전송 처리등을 수행하지 않음. (이렇게 목적지만 단순히 지정해서 전송하는 데이터를 데이터그램이라고 함)
			- TCP 프로토콜 : 헤더에 포트번호가 부여되어 있고, 오류 검사/확인/재전송 처리등의 과정도 규정되어 있으며 세션 통신을 수행하는 기능도 있음. 따라서 헤더에는 이 과정에 필요한 정보(시퀀스 번호, 응답 확인 번호 등)가 부여 됨.
